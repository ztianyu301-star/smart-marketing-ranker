# 简历描述建议

以下表述只覆盖这个项目里**真实存在**的能力点，适合你基于实际掌握情况做裁剪：

1. 设计并实现面向智能营销场景的多任务排序系统，基于用户、商品、查询与上下文特征联合建模 CTR/CVR，支持候选集重排与业务分数优化。
2. 搭建传统机器学习与深度学习双路线方案：使用 Logistic Regression 建立可解释基线，使用 PyTorch 实现 Wide & Deep 多任务排序模型，完成离线效果对比。
3. 构建完整工程流水线，包括合成样本生成、特征工程、模型训练、AUC/LogLoss/NDCG 评估、模型持久化与 FastAPI 在线推理接口。
4. 在模拟广告推荐场景中引入出价、价格、商家质量、文本匹配度、类目相关性等特征，实现兼顾点击率、转化率和业务收益的排序策略。

## 面试可讲的技术点

- 为什么要拆分 CTR 与 CVR 两个任务，以及多任务学习的收益
- 为什么需要业务分数而不是只看点击率
- Logistic Regression 与深度模型各自适合什么场景
- NDCG@10 在排序问题里比单纯准确率更有意义的原因
- 如何处理冷启动、类别稀疏、特征漂移和线上线下一致性
